# 인앱 결제 스토어 요구사항 및 세금 처리

## 스토어별 필요 항목

### Google Play (Android)
| 항목 | 필요 여부 |
|------|-----------|
| 결제 프로필 생성 | 필수 (Google Payments Center) |
| 신분증/여권 | 본인 확인용 필요할 수 있음 |
| 사업자등록번호 | **개인도 가능** (개인/사업자 선택) |
| 은행 계좌 | 수익 정산용 |
| 세금 정보 | 세금 양식 제출 필요 |

### Apple (iOS)
| 항목 | 필요 여부 |
|------|-----------|
| Apple Developer 계정 | **$99/년** |
| 유료 앱 계약 동의 | App Store Connect에서 필수 |
| 은행 계좌 + 세금 정보 | 필수 |
| 사업자등록번호 | **개인도 가능** |

**핵심:** 둘 다 **개인 자격으로 가능**. 사업자등록 없이 개인 신분증 + 은행 계좌만 있으면 됨.


## 수익 발생 시 세금 처리

### 개인 자격 시작 → 수익 발생 시

| 단계 | 내용 |
|------|------|
| 수익 소액 | 종합소득세 신고 시 **기타소득**으로 신고 |
| 수익 증가 | **사업자등록** 후 사업소득으로 전환 |
| 전환 시점 | 연 수익 수백만원 이상이면 사업자등록 권장 |

### 실제 순서
1. 개인으로 시작 → 앱 출시
2. 수익 발생하면 **5월 종합소득세** 신고 (홈택스)
3. 수익이 커지면 사업자등록 → 스토어 계정을 사업자로 전환

Google/Apple 모두 **개인 → 사업자 전환이 가능**하므로 처음부터 사업자를 만들 필요 없음.
수익이 생긴 후에 세무사 상담받는 게 효율적.


## 수익 수령 통화 (원화 vs 달러)

### 방법
| 방법 | 설명 |
|------|------|
| 외화(달러) 통장 | 국내 은행에서 **USD 외화예금 계좌** 개설 → 해당 계좌 등록 |
| 원화 통장 | 일반 원화 통장 등록 → 입금 시 **자동 원화 환전** |

### 달러 수령 장점
- 환율 유리할 때 직접 환전 가능
- 환전 수수료 절약 (환율 우대 적용 시)

### 스토어별 통화 지원
- **Google Play**: 통화 설정에서 USD 선택 + 달러 통장 등록 가능
- **Apple**: 기본적으로 해당 국가 통화(KRW)로 지급, USD 수령은 미국 계정 필요

### 달러 통장 개설
- 국내 은행(신한, 우리 등) 앱에서 바로 개설 가능
- 별도 서류 불필요, 비대면 개설 가능


## 인앱 결제 연동 진행 순서

1. Google Play Console에서 구독 상품 `anbu_yearly` 등록
2. Flutter `in_app_purchase` 패키지 추가
3. 구매/복원 로직 구현
4. 서버 영수증 검증 API 연동 (`POST /api/v1/subscription/verify`)
5. iOS는 Apple Developer 계정 확보 후 동일 진행
